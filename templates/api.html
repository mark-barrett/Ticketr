Ticketr API. For Developers or Anyone.


Ticketr API - Documentation

The Tickter API allows the Ticketr app and other developers to communicate with the Ticketr database to get certain information. The API returns data in JSON format. The API comes with 4 API handles which are the following:

  • My Tickets - Providing a users username and password will return a list of the users tickets. The Ticketr app will use this to show QR codes on the Android app.
    Usage:
    http://178.62.41.17/api/tickets
    Posting: username
    Posting: password

  • My Events - Providing a users username and password will return a list of the users events. The Ticketr app will use this to show an event organiser what events he/she has.
    Usage:
    http://178.62.41.17/api/events
    Posting: username
    Posting: password

  • Authentication - Providing a users username and password will return a message either authenticating the user as valid or reutrning an error.
    Usage:
    http://178.62.41.17/api/authenticate
    Posting: username
    Posting: password

  • Ticket Validation - Providing the Event owners ID for authentication, the order number, and the name and order code will be returned.
    Usage:
    http://178.62.41.17/api/validate-ticket
    Posting: event_owner_id
    Posting: order_number
